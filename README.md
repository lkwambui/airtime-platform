# Kredo ChapChap

Admin airtime sales platform with M-Pesa integration.

## Local Development

### Backend
```bash
cd backend
npm install
npm run dev
```

### Admin Panel
```bash
cd frontend/admin
npm install
npm run dev
```

### Client App
```bash
cd frontend/client
npm install
npm run dev
```

## Database Setup

This project uses **PostgreSQL**.

1. Create a PostgreSQL database (locally or on a hosted provider like [Neon](https://neon.tech/) or [Render](https://render.com/))
2. Run migrations in order from `backend/src/database/migrations/`
3. Run seeds from `backend/src/database/seeds/` to create the default admin user

```bash
psql -U <user> -d <database> -f backend/src/database/migrations/001_create_admins.sql
psql -U <user> -d <database> -f backend/src/database/migrations/002_create_settings.sql
psql -U <user> -d <database> -f backend/src/database/migrations/003_create_transactions.sql
psql -U <user> -d <database> -f backend/src/database/migrations/008_add_jobs_and_devices.sql
psql -U <user> -d <database> -f backend/src/database/seeds/admin_seed.sql
```

## Production DB Patching (Neon)

Use this idempotent migration when production schema needs to be aligned safely:

```bash
psql "<neon-connection-string>" -v ON_ERROR_STOP=1 -f backend/src/database/migrations/009_align_production_schema_idempotent.sql
```

Notes:
- Safe to re-run (`IF NOT EXISTS` / guarded `ALTER` statements).
- If a database URL is exposed in logs/chat, rotate the Neon password/secret immediately.

## Deployment on Render

Deploy all three services using the included `render.yaml` blueprint.

### Steps

1. Push your code to GitHub.
2. Go to [Render Dashboard](https://dashboard.render.com/) → **New → Blueprint**.
3. Connect your GitHub repo and select the branch.
4. Render will detect `render.yaml` and provision all three services automatically.

### Environment Variables to Set Manually

The following variables have `sync: false` in `render.yaml` and **must be set manually** in the Render dashboard for each service:

**Backend (`airtime-backend`):**

| Variable | Where to get it |
|---|---|
| `DB_HOST` | Your Render PostgreSQL or external DB hostname |
| `DB_USER` | Your database user |
| `DB_PASSWORD` | Your database password |
| `DB_NAME` | Your database name |
| `ADMIN_PIN` | Choose a secure PIN |
| `MPESA_CONSUMER_KEY` | [Safaricom Developer Portal](https://developer.safaricom.co.ke/) |
| `MPESA_CONSUMER_SECRET` | Safaricom Developer Portal |
| `MPESA_SHORTCODE` | Your registered M-Pesa business number |
| `MPESA_TILL` | Your M-Pesa till number |
| `MPESA_PASSKEY` | Safaricom Lipa Na M-Pesa passkey |
| `MPESA_CALLBACK_URL` | `https://airtime-backend.onrender.com/api/mpesa/callback` |
| `AUTOBUNDLES_CLIENT_ID` | AutoBundles account |
| `AUTOBUNDLES_CLIENT_SECRET` | AutoBundles account |
| `DEVICE_API_KEY` | Generate a strong random secret for Android device auth |

> `JWT_SECRET` is auto-generated by Render — no action needed.

### Post-deploy

After all services are live, run your migrations against the Render PostgreSQL database:

```bash
psql <render-postgres-connection-string> -f backend/src/database/migrations/001_create_admins.sql
# ... repeat for each migration file in order
```

Or connect via `psql` using the **External Database URL** from the Render dashboard.
